## Goal

Replace the current `wpa_documents` DB-backed approach with a live file tree that browses the WPA file server directly. The Docs tab on both client and lead panels should show the actual folder contents from `https://whitepine-tech.com/files/` in real time — no manual record management.

---

## File Server Details

**Base URL:** `https://whitepine-tech.com/files/`
**Auth:** `X-Api-Key` header
**API key:** store as `VITE_FILE_SERVER_API_KEY` in `.env.local` — get the value from Christopher
**Directory listing endpoint:** `GET /files/<path>/`
**Response format:**
```json
{
  "path": "/Clients/Active/Dog-Zen/",
  "entries": [
    { "name": "context.md", "type": "file", "size": 1234, "modified": "2026-03-14T09:30:00.000Z" },
    { "name": "Analytics", "type": "directory", "size": 4096, "modified": "2026-03-12T10:00:00.000Z" }
  ]
}
```
**File download:** `GET /files/<path>/<filename>` — direct link, opens/downloads the file.

**Folder structure on the server:**
```
WhitePineAgency/
├── Clients/
│   ├── Active/
│   │   ├── Dog-Zen/
│   │   └── Harveys-on-the-Green/
│   └── Leads/
│       ├── Goodrich-Plumbing-Idaho/
│       ├── Janets-Cafe/
│       ├── Native-Landscape-Services/
│       └── ...
├── Knowledge/
└── Research/
```

**All `folder_path` values are relative to the file server root**, so they include the `WhitePineAgency/` prefix:
- Active client example: `WhitePineAgency/Clients/Active/Dog-Zen`
- Lead example: `WhitePineAgency/Clients/Leads/Goodrich-Plumbing-Idaho`

## The Mapping Problem (Critical — Solve First)

The file server uses human-readable folder names (`Dog-Zen`, `Goodrich-Plumbing-Idaho`). The CC database has no field linking a client/business record to its folder name. We need to add one.

### Solution: Add `folder_path` column to `wpa_clients` and `wpa_businesses`

**Migration:** Create `supabase/migrations/YYYYMMDDHHMMSS_add_folder_path.sql`:

```sql
-- Add folder_path to wpa_clients (active clients)
ALTER TABLE wpa.wpa_clients ADD COLUMN IF NOT EXISTS folder_path TEXT;

-- Add folder_path to wpa_businesses (leads)
ALTER TABLE wpa.wpa_businesses ADD COLUMN IF NOT EXISTS folder_path TEXT;

COMMENT ON COLUMN wpa.wpa_clients.folder_path IS 'File server path relative to /files/ root, e.g. Clients/Active/Dog-Zen';
COMMENT ON COLUMN wpa.wpa_businesses.folder_path IS 'File server path relative to /files/ root, e.g. Clients/Leads/Goodrich-Plumbing-Idaho';
```

**Seed known values** in the same migration or a follow-up:
```sql
UPDATE wpa.wpa_clients SET folder_path = 'WhitePineAgency/Clients/Active/Dog-Zen' WHERE name ILIKE '%dog zen%';
UPDATE wpa.wpa_clients SET folder_path = 'WhitePineAgency/Clients/Active/Harveys-on-the-Green' WHERE name ILIKE '%harvey%';

UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Goodrich-Plumbing-Idaho' WHERE name ILIKE '%goodrich%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Janets-Cafe' WHERE name ILIKE '%janet%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Native-Landscape-Services' WHERE name ILIKE '%native landscape%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Everything-Irrigation' WHERE name ILIKE '%everything irrigation%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/M-2-Landscape-Contractors-Inc' WHERE name ILIKE '%m-2%' OR name ILIKE '%m2 landscape%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Heritage-Plumbing-LLC' WHERE name ILIKE '%heritage plumbing%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Sarahs-Bagel-Cafe' WHERE name ILIKE '%sarah%bagel%';
UPDATE wpa.wpa_businesses SET folder_path = 'WhitePineAgency/Clients/Leads/Seasonal-Lawncare' WHERE name ILIKE '%seasonal%';
```

**Note on EverythingIrrigation:** Previously had two folders — merged and standardized to `Everything-Irrigation`.

`folder_path` is set manually when a lead folder is created. It's nullable — if null, the Docs tab shows an empty state with a note that no folder exists yet.

## Folder Naming Convention (Enforced Going Forward)

All client and lead folders use **Title-Kebab-Case**:
- Each word capitalized, words separated by hyphens
- Examples: `Dog-Zen`, `Harveys-on-the-Green`, `Goodrich-Plumbing-Idaho`, `Janets-Cafe`
- No CamelCase (`JanetsCafe` ✗), no lowercase (`goodrich-plumbing` ✗), no spaces
- When creating a new lead folder, derive the name from the business name: drop punctuation, capitalize each word, join with hyphens
- Set `folder_path` in CC at the same time the folder is created on beefy

OC Bob is aware of this rule and will apply it when creating lead folders.
